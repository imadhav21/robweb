Discussing the Implementation of Breath-First Search

Breadth-first search in general is meant to always find the shortest path, as it makes sure to keep the parent node and distance from the parent node in check. And if there is essentially a loop, it will compare the distances, or costs, and make sure to update the path so it is the shortest path.

However, in our implementation of the code, there was some bug where it did not always choose the best path, but had sort of a zig-zag motion at times, as you can see below, as well as in the IRL videos of the robot. However, this did not hinder the overall performance of the robot, and the robot still traversed through the maze without hitting any obstacles.

The strengths and Weaknesses of this Implementation

I think that one of the biggest limitations with graph search, in general, is that it seems to take more processing power than the distance transform, as it has to keep track of all of the nodes in the graph, the parents, and the cost, as well as updating the costs as necessary.

While this is more intensive than using the potential field, BFS and graph search are way way better than using the potential field. Namely, it did not run into any obstacles because of local minimums, and it is able to find the goal each and every time and drive almost perfectly to it. While in the potential field there was a lot of tuning to be done concerning how much the repulsive field should repel and the fact that the potential field would break if it was in a maze, Breath-First Search handles it like a champ, and it is able to traverse through any maze, as seen above in the video.

One thing to note is that A-Star would defintely be a more optimal algorithm to use, as it is more efficient in how it searches the graph and prioritizes the nodes as it checks them.

For the Breath-First Search algorithm we implemented, we could definitely try to fix the issue of the robot not choosing the shortest path. This is due to how the robot is updating the costs of the neighboring cells, and that the distances are not being properly updated along with the neighbors. This would help to make the robot travel slightly faster.
